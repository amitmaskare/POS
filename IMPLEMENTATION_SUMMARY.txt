TESTING INSTRUCTIONS
================================================================================

1. Run the database migration:
   cd Backend
   node database_files/development/run_migration.js

2. Restart backend server:
   npm run dev

3. Test Super Admin Login:
   - Login as super admin (user_id: 1000, password: 123456)
   - Go to Users page
   - Verify you can see ALL users including yourself

4. Test Store Admin Creation:
   - As super admin, create a new store admin
   - Note the store_id assigned (e.g., STORE1001)

5. Test Store Admin Login:
   - Login as the store admin you just created
   - Go to Users page
   - Verify you CANNOT see super admin in the list
   - Verify you can see only users from your store

6. Test Sub-Admin Creation:
   - As store admin, create a new sub-admin
   - Verify sub-admin appears in user list
   - Verify sub-admin has same store_id

7. Test Cashier Creation & Device Binding:
   - As store admin, create a cashier
   - Note the cashier login credentials
   - Logout and login as cashier from a device
   - Verify device_locked = 1 in user list (view as admin)
   - Verify "Device Locked" chip shows with device ID tooltip

8. Test Device Unbind:
   - As store admin, view user list
   - Find cashier with "Device Locked" status
   - Click "Unbind Device" button
   - Confirm the action
   - Verify chip changes to "Not Bound"
   - Verify cashier is logged out (if active)

9. Test Cross-Store Isolation:
   - Create two store admins (Store A and Store B)
   - Login as Store A admin, create cashier
   - Login as Store B admin
   - Verify you CANNOT see Store A admin or cashier

SECURITY FEATURES
================================================================================

1. Store Isolation: Admins cannot access data from other stores
2. Super Admin Hidden: Regular admins don't know super admin exists
3. Device Binding: Cashiers locked to first login device
4. Immediate Logout: Device unbind invalidates active sessions
5. Created By Tracking: Audit trail of user creation
6. JWT Token Security: store_id embedded in token (tamper-proof)
